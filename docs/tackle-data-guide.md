# タックルガイド データ管理ルール

タックルガイドページ（`/tackle/`）のデータ管理に関するルールをまとめたドキュメントです。

---

## ディレクトリ構成

```
data/tackle/
  categories.json          ← カテゴリ一覧（全カテゴリ共通）
  rod.json                 ← ロッドカテゴリのアイテム一覧
  reel.json
  line.json
  terminal.json
  accessories.json
  extras.json
  rod/
    casting-rod.md         ← 投げ竿の詳細記事（Markdown）
    iso-rod.md             ← ← 今後追加する場合はここに
  reel/
    spinning.md            ← ← 今後追加する場合はここに
  ...
```

---

## ファイルの役割分担

| ファイル | 役割 | 主な用途 |
|---|---|---|
| `categories.json` | カテゴリ定義 | カテゴリ一覧ページの表示 |
| `{カテゴリ}.json` | アイテムのメタデータ＋アフィリエイトデータ | 一覧カード・商品リンク |
| `{カテゴリ}/{slug}.md` | アイテムの詳細記事（Markdown） | 詳細ページの本文 |

**JSONは「短いメタデータ」と「商品リンクデータ」だけ持つ。本文はMarkdownファイルが担当する。**

---

## 各ファイルのスキーマ

### `categories.json`

```json
[
  {
    "slug": "rod",
    "name": "ロッド（竿）",
    "icon": "🎣",
    "summary": "カテゴリ一覧ページに表示する1〜2行の説明",
    "intro": "カテゴリページ上部に表示するより詳しい説明文",
    "coming_soon": false   // 省略可。trueの場合はリンクを無効化
  }
]
```

### `{カテゴリ}.json`（例: `rod.json`）

```json
[
  {
    "slug": "casting-rod",           // URLのslug。Markdownファイル名と一致させる
    "name": "投げ竿",                // 表示名
    "summary": "一覧カードに出る1〜2行の説明",
    "body": "短い説明文（Markdownファイルがない場合のフォールバック用）",
    "suitable_for": "向いている釣り方（Markdownファイルがない場合のフォールバック用）",
    "beginner_tip": "初心者へのアドバイス（Markdownファイルがない場合のフォールバック用）",
    "affiliate_links": [],           // 旧フィールド。現在は affiliate_slots を使う
    "affiliate_slots": [             // 商品リンクデータ（後述）
      [{"name": "商品名", "url": "...", "price": "3,000円", "note": "補足"}],
      [],
      []
    ]
  }
]
```

### `{カテゴリ}/{slug}.md`（例: `rod/casting-rod.md`）

通常のMarkdownファイル。記事本文全体を書く。詳細は後述。

---

## Markdownファイルの書き方

### 基本ルール

- 見出しは `##`（大セクション）と `###`（小セクション）で構成する
- 比較・スペック表は必ずMarkdownテーブルで書く
- 強調は `**テキスト**`、引用ブロックは `>`
- 箇条書きは `-`
- セクション区切りは `---`

### アフィリエイトリンクの挿入位置

商品リンクを表示したい位置に以下のコメントを挿入する：

```
<!-- affiliate: 1 -->
```

- 番号は **記事の上から順に `1`, `2`, `3` ...** と連番
- 前後に空行を入れる
- 番号は `{カテゴリ}.json` の `affiliate_slots` 配列のインデックスに対応（1始まり）
- 商品データが空（`[]`）のスロットは何も表示されない。**マーカーだけ先に書いておいてよい**

### 記事構成の目安

```markdown
# ○○完全ガイド｜〜のリード文

導入段落

---

## 基本知識セクション
### 小見出し
説明・表

---

## 【入門クラス】予算〜

> ターゲット読者の一言

### こんな方におすすめ
### 性能と特徴
### 代表的なモデル（例）

<!-- affiliate: 1 -->

## 【ミドルクラス】予算〜
（同様の構成）

<!-- affiliate: 2 -->

## 【ハイエンドクラス】予算〜
（同様の構成）

<!-- affiliate: 3 -->

## まとめ

比較表・総評

---

## 合わせて揃えたいもの
## よくある質問（FAQ）
```

※ カテゴリによって入門/ミドル/ハイエンドの区分でなくても構わない。  
　 アクセサリー類なら商品種別ごとにセクションを分けてマーカーを入れればよい。

---

## アフィリエイトスロットの管理

### `affiliate_slots` フィールドの構造

```json
"affiliate_slots": [
  // スロット1（<!-- affiliate: 1 --> に対応）
  [
    {
      "name": "商品名（ブランド＋型番が望ましい）",
      "url": "https://...",        // アフィリエイトリンク
      "price": "3,000〜5,000円",  // 実売価格帯（省略可）
      "note": "入門に最適"         // 補足コメント（省略可）
      // ※ "image" フィールドは現在未対応。書いても表示されない。
      //    将来 Amazon PA-API 等と連携する際に実装予定。
    }
  ],
  // スロット2（<!-- affiliate: 2 --> に対応）
  [],   // 空配列 = 何も表示しない（準備中でもOK）
  // スロット3（<!-- affiliate: 3 --> に対応）
  []
]
```

### 商品を追加・更新するとき

1. `{カテゴリ}.json` の該当アイテムの `affiliate_slots` を編集する
2. Markdownファイルは変更不要
3. スロット番号（`<!-- affiliate: N -->`）はMarkdownファイルの番号と一致させる

### 商品を削除するとき

該当スロットを空配列 `[]` に戻すだけでよい（マーカーはMarkdownに残しておいてよい）。

---

## Markdownファイルがない場合（フォールバック）

`{カテゴリ}.json` の `body` フィールドの内容が表示される（改行を `<br>` に変換）。  
既存の短い説明文はそのままで動作する。

---

## 新しいアイテムを追加するとき

1. `{カテゴリ}.json` に新しいオブジェクトを追加する（`slug`, `name`, `summary` は必須）
2. 詳細記事を書く場合は `{カテゴリ}/{slug}.md` を作成する
3. ファイル名（slug）は `{カテゴリ}.json` の `slug` と必ず一致させる

---

## 新しいカテゴリを追加するとき

1. `categories.json` に新しいオブジェクトを追加する
2. `{カテゴリ}.json` を新規作成する
3. 開発者に `/tackle/{カテゴリ}/` のルーティング追加を依頼する（`app/main.py` は変更不要。データファイルを置くだけで自動認識される）

---

## 注意事項

- **`slug` は変更しない。** URLが変わり、既存リンクが切れる
- `affiliate_slots` の配列の順番は変更しない（Markdownのマーカー番号とずれる）
- `url` フィールドにはアフィリエイトタグ付きのURLを入れる
- 商品リンクのない状態（`[]`）でもページは正常に表示される
